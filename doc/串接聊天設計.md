# 配對聊天 API
- **URL**: `https://tlk.io/<channel_id>` (頻道 ID 上限 30 字元)
- **嵌入網頁碼**: `<div id="tlkio" data-channel="<channel_id>" data-nickname="<nickname>" style="width:100%;height:400px;"></div><script async src="http://tlk.io/embed.js" type="text/javascript"></script>`

## 聊天室識別碼設計

### 1. `tlk.io` 聊天室識別碼如何利用？
我們將使用系統生成的 `matchTicketId`（每日配對票券的唯一 ID）直接作為 `tlk.io` 的聊天室識別碼 (`tlkChannelId`)。

### 2. 如何分配識別碼以維持對話延續性？
因為每個配對事件都有一個獨一無二的 `matchTicketId`，所以每個配對都會創建一個專屬的聊天室。只要 `matchTicketId` 不變，使用者就可以回到同一個聊天室，從而維持了對話的延續性。

**結論**: `matchTicketId` = `tlk.io` 聊天室 ID。

## 暱稱設定
為了讓使用者在進入聊天室後能直接顯示暱稱，前端在嵌入 `tlk.io` 視窗時，需在 `<div id="tlkio">` 標籤中加入 `data-nickname` 屬性。例如：`data-nickname="使用者暱稱"`。此暱稱應來自使用者在啟動聊天前所輸入或系統預設的暱稱。
